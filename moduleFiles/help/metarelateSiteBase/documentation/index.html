Metarelate Terminology Server Help


The help section is organised in the following files

  • Deployment and Overview explains the basics of the architecture of the system and its intended usage.
  • HOW-TO change the default configuration explains how to tweak basic systems properties as file directories
  • HOW-TO ingest a terminology explains what the system expect and how it can be provided
  • HOW-TO configure the process explains how to configure possible processes (e.g. validation, obsoletion) and relative state transitions
  • HOW-TO write a template presents a simple explanation of the template language used in html and latex templates.
  • Technical Notes presents notes on the state of the code, limitations and possible improvements.
  • A link to the Javadocs is available here.
  • The configuration language is broken down in three language spaces and it is represented through "ontologies" (weak conceptualization):
    • config comprises classes and properties that drive the configuration of the system and its properties. The corresponding OWL/RDF file can be found here.
    • core comprises classes and properties used within the system. The cooresponding OWL/RDF file can be fond here.
    • process comprises classses and properties to support states and transitions. The corresponding OWL/RDF file can be found here.
