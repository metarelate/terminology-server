How To publish a terminology model

Before following these instruction, make sure the system is installed. Refer to HOW-TO Install for further details.
Publishing a terminology model is possible only once a terminology model has been created. Information on how the terminology publication fits within the overall project is provided in the Terminology Manager platform design document.
Publishing a terminology as web resources can be done via the metMap command, as detailed below:

1) Make sure the classpath contains all required libs

Example command:

export CLASSPATH=$CLASSPATH:/home/test/MetOfficeTermRegistry/distrib/WMOTermReg.jar:/home/test/MetOfficeTermRegistry/libs/*

2) Execute MetMap

Example command:
java uk.gov.metoffice.terminology.commandline.MetMap 
-d 
-ow 
-o web 
-p file:////home/test/MetOfficeTermRegistry/configurationFiles/metOfficeDeploymentGeneric.ttl 
-m tdb /home/test/DB/MetOffice 
-f file:////home/test/MetOfficeTermRegistry/configurationFiles/metOfficeDeploymentGeneric.ttl 
file:////home/test/MetOfficeTermRegistry/configurationFiles/metOfficeDeploymentServer.ttl 
file:////home/test/MetOfficeTermRegistry/configurationFiles/genericPropertyAttributes.ttl
Note: the specific syntax of the above command is currently changing. Explanations will be provided via --help.

Explanation:
-d              sets debug on (verbose logging)
-ow           sets overwrite on: the system will overwrite a file if this is already present (it will stop otherwise)
-o web      species that metMap should produce a web representation of codes as files
-p              specifies a file whose prefixes are used as prefixes in the relevant RDF serializations (RDF/XML, Turtle)
-tdb       specifies the location (and type) of the triplestore.
-f               specifies a set of files with further configurations.  All files are read and merged in a unique input processed by the system and how the configuration is cut into files is an arbitrary choice. Details on the configuration possibilities are presented in the Guide to Configuration Files.

Compact commands

To simplify portability some macro command is provided (in the scripts directory of the distribution).

publishRoot

Example command:
./publishRoot.py
The script relies on a fixed directory structure and executes metMap with "standards" argument sets, and site specific arguments extracted from the file publishenvironment.txt (the format is parameter ONE_SPACE value).
